<summary><h2>Detailed Tech Stack</h2></summary>

### Frontend

- Building modern, high-performance interfaces using **Next.js 16**, **React 19**, and **TypeScript (strict mode)**
- Leveraging **Vite** for fast development and optimized builds
- Styling with **Tailwind CSS**, **Mantine UI**, and **Styled Components**
- Implementing smooth UI/UX with **Framer Motion** and responsive design patterns
- State management using **Redux Toolkit**
- Schema validation and type-safe data handling with **Zod**
- Real-time features via **Socket.IO client**
- Rich text editing with **Tiptap**
- Testing with **Vitest** and **React Testing Library**

---

### Backend — Node.js

- Designing scalable backend systems with **NestJS** and **Node.js** using **TypeScript (strict)**
- Applying architectural patterns: **CQRS**, **Event-Driven Architecture**, and **Hexagonal Architecture**
- Building RESTful APIs with strong typing, validation, and modular service structure
- Working with **PostgreSQL** using **TypeORM**
- Event streaming and async communication via **Kafka (KRaft mode)**
- Real-time communication with **Socket.IO gateways**
- Authentication and authorization with **JWT**, **OAuth (GitHub, Google, Facebook)**, and **MFA**
- Integrating external services: **Stripe (payments)**, **AWS S3**, **Nodemailer**, **Sharp**
- API documentation using **Swagger / OpenAPI**

---

### Backend — Go

- Developing lightweight, high-performance microservices in **Go (1.26)**
- Using **Chi router** for minimal and efficient HTTP routing
- Working with **PostgreSQL** via **pgx**
- Kafka integration with **kafka-go**
- File storage and object handling via **AWS SDK v2 (S3 / MinIO)**
- Focus on simplicity, performance, and clear service boundaries

---

### Tooling & DevOps

- Managing monorepo architecture with **Yarn Berry (PnP)** and **Workspaces**
- Containerization using **Docker** and **Docker Compose**
- Orchestrating services locally with **Kubernetes (Colima-based cluster)**
- Implementing CI/CD pipelines using **GitHub Actions** (per-service workflows)
- Maintaining code quality with **ESLint**, **Prettier**, and strict TypeScript rules
- Local infrastructure tooling: **MinIO (S3)**, **Mailpit (SMTP)**, **Kafka UI**

---

### Cloud & Infrastructure

- Deploying and managing services on **AWS**:
  - EC2 (compute)
  - S3 (file storage)
  - Lambda (serverless)
  - API Gateway
  - DynamoDB
- Using **Cloudflare** for media delivery (Images, Stream)
- Deploying applications via **Vercel**, **Heroku**, and **Render**

---

### Architecture

- Designing and maintaining a **microservices architecture**:
  - backend
  - payment-service
  - file-service
- Using **shared contracts** for type-safe communication (HTTP + Kafka DTOs)
- Implementing **internal service authentication** (x-service-token)
- Event-driven communication via **Kafka topics** (files.*, payments.*, blog.*)
- Strong focus on modularity, scalability, and clear service boundaries
